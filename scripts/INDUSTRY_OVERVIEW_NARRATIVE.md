# 業界俯瞰レポート — ナラティブ構成設計

## 設計原則

1. **生データ → カテゴリ定義 → 指標定義 → 検証 → 分析 → 知見** の一方向フロー
2. 前方参照は避け、常に既出の事実・定義のみに立脚する
3. 各セクション冒頭で「何を示すか」「なぜここで必要か」を1文で述べる
4. 定量的主張には必ずデータ根拠（チャート番号/数値）を示す
5. 独自指標は「なぜ必要か → どう算出するか → どう解釈するか」の3段階で導入する
6. insight box の主張は、可能な限りプログラムで計算した値から生成する（ハードコードしない）

## 全体構成

### §0 Pipeline Summary
- パイプライン実行結果の基本統計（人数・作品数・グラフ規模）

### §1 分析手法と定義
- **1.1 データソース**: AniList / MADB の性質・制約
- **1.2 分析対象期間**: パラメータと根拠
- **1.3 キャリアステージ定義**: 6段階、妥当性は§3 V-2で検証
- **1.4 引退の操作的定義**: 3年閾値、妥当性は§3 V-1で検証
- **1.5 時代特性と補正**: 比率計算、カットオフ
- **1.6 定量指標の定義** (NEW):
  - なぜ独自指標が必要か（視聴者評価を使わない理由）
  - 7指標の表: BiRank, Person FE, Patronage, AWCC, Dormancy, IV Score, composite_value
  - IV Score のλ重み: PCA PC1 loadings（データ駆動、分散説明率も開示）
  - 解釈上の注意: 高IV ≠「上手い」
  - 検証ポインタ: V-4c, Chart D
- **1.7 レポート構成ガイド**: 依存関係つき読み順

### §2 生データ概観: 業界規模の推移
- Time Series 4-panel + Dual-axis
- §1.2の根拠を実証（1970年以前の不安定性、2000年代の急成長）

### §3 前提検証
- **V-1**: 引退閾値3年の検証
  - insight boxは `gap_stats_rows` から動的にP95≤3の職種数を計算
  - 例外（P95>3の職種）も数値で明示
- **V-2**: ステージ分類の検証
  - **Spearman ρ を計算・表示**（相関の統計的根拠）
  - K-Means(K=3)一致率: **stageをfeatureに含めない**（循環を回避）
- **V-3**: 引退時キャリア年数分布（職能依存性の確認）
- **V-4**: Sankey図（遷移パスの可視化）
- **V-4b**: アニメーター出身 vs 非アニメーター出身 監督の分離
- **V-4c**: キャリア長 × composite_value 散布図
  - **Spearman ρ を各typで計算・表示**
  - 循環論法リスクを明記（BiRank→グラフ→キャリア長）
- **V-5**: 遷移確率行列・外部流出率

### §4 人材フロー分析
- 接続文: §3で検証済みの定義に基づく
- **Chart D**: 期待能力の計算から **anime.score を完全排除**
  - 代替: composite_value (BiRank由来、§1.6参照)
  - 3成分すべて構造的指標のみ
  - `pid_latest_year.get(pid, RELIABLE_MAX_YEAR)` バグ修正
- **Chart F**: IV Score合計による価値流出入
  - **時代間比較の注意事項を追加**（ネットワーク規模の拡大効果）
- **Chart J**: K-Meansクラスタ別セグメント

### §5 構造分析
- **5.1 需給構造**: format別年次推移（接続文で§4との関係を明示）
- **5.2 季節パターン**:
  - Chart A: anime.score → **「視聴者評価（参考情報）」に改称**、スコアリング不使用を明記
- **5.3 世代別キャリア軌跡**: §4からの再集計（接続文あり）

### §6 総括
- §2-§5の全知見を統合
- データ駆動の知見（未検証の主張を削除）
- 活用ガイド、今後の可能性

## 残存する既知の制約

- **composite_value検証の循環性**: BiRankはグラフ構造から計算されるため、キャリア長との相関は構造的に生じうる。独立した外部検証（受賞歴等）は今後の課題
- **V-3とV-5の順序**: ナラティブ設計ではV-3=遷移確率行列だが、実装ではV-3=引退年数分布。番号を実装に合わせて維持
- **insights_report.json**: 自動生成知見の出所が未説明（パイプラインのphase 9で生成）
- **§5-§4の分析的接続**: 接続文で言及するが、需給とフローの定量的関連分析（相関、Granger因果）は未実装
