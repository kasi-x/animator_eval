# 議連 / 政策研究会経由のアクセス Protocol

## 文書目的

本文書は、Animetor Eval が議員連盟 (議連) および政策研究会経由で政策決定者にアクセスするための体系的な protocol を定める。labor-first スタンス (STANCE.md §1) を維持しつつ、**政治的中立性と超党派性を確保する**ことを最優先とする。

政策経路の重み: STANCE.md §2 において 政策提言 32% と定められている。議連経由アクセスは、当該 32% の実装経路の一つであり、並行する ministry outreach (別 doc) と並走する。

最終更新: 2026-05-13

- **知的財産戦略推進事務局 (内閣府)** — 非議連だが準公的フォーラム。知財・コンテンツ戦略に関わる政策立案グループ

- 内閣府・衆参両院事務局の議連リスト
- 業界新聞 (アニメ!アニメ! / コミックナタリー 等) の記事
- JAniCA 等業界団体からの情報提供

**Animetor Eval 運営者が確認すべき項目**（実装時）:

- [ ] 議連の最新の会長・事務局長氏名
- [ ] 事務局連絡先 (メール / 電話)
- [ ] 直近の勉強会日程・議題
- [ ] 各議連の党派構成 (超党派比率の確認)
- [ ] 過去の招聘者・ゲスト講師

## 2. 政治的中立性の定義と確保方法

### 2.1 中立性の原則

**定義**: Animetor Eval は、特定の政党・政治的立場を支持・反対しない。ただし、政策内容（労働環境改善・機会格差是正・クレジット公表標準化等）には**労働者寄り立場 (labor-first)** を取る。

- **禁止**: 「○○党の△△議員の主張を支持する」「特定政党のために有利な分析」
- **許可**: 「クレジット公表率の低さは業界的課題である」「ジェンダー機会格差是正は政策が重要」

### 2.2 超党派アクセスの実装ルール

1. **同時・段階的提供の原則**

   同じ内容の brief / データを、政党横断的に提供する。特定の一派に先行提供しない。
   
   例外: 議連の主催者からの招待があった場合、その議連に対しては直接データ提供可能。但し事後に他の議連にも同等の情報開示をする（1-2 周間遅延は容認）。

2. **党派構成の確認**

   アクセス前に、対象議連の党派構成 (自民 / 立民 / 公明 / 維新等) を調査し、「特定政党に極端に偏った議連」との判定が出た場合は、ministry outreach に経路を切り替える（ただし議員個人からの招待があれば対応）。

3. **個別議員への特定政治活動への利用拒否**

   個別議員が Animetor Eval データを「特定政党の政治活動」（選挙演説、党派的主張等）に使用したいという要求があった場合、Animetor Eval は以下の対応をする:
   
   - 1 回目: 「当プロジェクトは政治的中立性を保つため、単一政党の政治活動への専属利用はできない。代わりに全議連に同等の情報を提供しており、そちらの枠組みでの使用をお願いしたい」と丁寧に説明
   - 2 回目以降: 招待・協力を慎重に検討し、利益相反の判定を実施

### 2.3 Crisis Communication — 党派的報道への対応

万が一、報道等により「Animetor Eval は特定政党に偏っている」と報じられた場合:

1. **事実確認と内部記録**

   - 当該発言・報道の全文を記録
   - Animetor Eval 側の実際のアクション (誰に何を提供したか、タイムライン) を整理

2. **公式コメント発表**

   例文（テンプレート）:

   > Animetor Eval は政治的に中立なデータ分析プロジェクトです。公開 briefs / データは全議連・政党横断的に提供しており、特定政党のために最適化されたものではありません。データの解釈や政策提言の決定は、各議員・政策担当者の判断であり、Animetor Eval の立場を代表するものではありません。

3. **対応機関への説明**

   - 業界団体 (JAniCA など協力関係にある組織)
   - 学位機関の指導教員 (COI disclosure)
   - 必要に応じて、誤報道を行ったメディアへの訂正申し入れ

## 3. アクセス段階フロー

以下は理想的なアクセス段階を示す。ただし、議員・議連の側からの打診があった場合は、段階をスキップして Step 3-4 から開始できる。

### Step 1: 公開情報による監視

**実施主体**: Animetor Eval 運営者 / 学振研究費の補助対象であれば学振が推奨する「研究成果の社会還元」枠組み内で実施

**アクション**:

- 各議連の公開ウェブサイト・議事録を定期的に閲覧
- 「アニメ」「クレジット」「労働」「フリーランス」等キーワードでの議事録検索
- 直近の勉強会テーマ・ゲスト講師の分析
- 各議連の会長・事務局長を identify

**出力**: 「議連アクセス候補リスト」(xls / json)

- 議連名 / 党派構成 / 主な関心領域 / 最新 meeting 内容 / 連絡先

### Step 2: 学位機関・業界団体経由の紹介（推奨）

**戦略的理由**: 直接的な政治アクセスは外部からの批判を受けやすい。学位機関の「研究成果公開」枠組みまたは業界団体 (JAniCA等) の推薦を経由することで、credibility を高める。

**実施チェックリスト**:

- [ ] 学位機関（機関の研究推進部門）に「議連への brief 提供予定」を事前報告
- [ ] 機関からの推薦状 / 紹介文を取得（可能な範囲で）
- [ ] JAniCA など協力機関があれば、その団体経由での紹介を要請

**メール案（議連事務局へ）**:

```
件名: アニメ産業の労働・構造分析データ提供のご提案

議員連盟 ○○ 事務局長 △△ 様

いつもお世話になっております。

[機関名] 博士課程の研究者として、アニメ業界の構造観察に関するデータ分析を行っております。
貴議連の「[直近の議題 / 関心領域]」に関連して、
業界のクレジット可視性・労働流動性に関する定量的知見をご提供できる可能性があります。

つきましては、brief 送付および 30 分程度のご面会の機会をいただけないでしょうか。

詳細は以下のとおりです。

- プロジェクト概要: [URL]
- 提供可能な分析: クレジット公表率、ジェンダー機会格差、中堅人材流出 等
- brief 形式: 2 ページ短縮版 + 詳細版オプション

政治的中立性を保つため、本データは全議連に並行して提供しております。

ご検討のほど、よろしくお願いいたします。

[署名]
[機関名・職位]
[メール / 連絡先]
```

### Step 3: 初回接触と Brief 送付

**実施者**: Animetor Eval 運営者 (最初のコンタクトは本人が行う。秘書への委任は事後的に可)

**準備物**:

- 2 ページ短縮版 brief (33_policy/01_short_form_brief.md 参照)
- 詳細 Policy brief (オプション)
- プロジェクト概要シート (1 ページ)
- 自己紹介とアフィリエーション確認書

**brief の内容要件**:

- 言語: JA + EN (短縮版は JA のみでも可)
- 構成: Findings / Key figures / Policy implications / Limitations / Contact
- forbidden vocab のクリア (scripts/report_generators/lint_vocab.py で自動チェック)
- Disclaimer 必須: 「本プロジェクトは労働者寄りの立場を取りますが、特定政党の政治活動を支持するものではありません」

**送付物の量的制限**:

- 初送付は brief のみ（詳細版は「必要に応じて」の位置づけ）
- PDF / 紙版双方の送付を検討（議員秘書は紙派が多い）

### Step 4: 面会・ヒアリング

**実施体制**:

- Animetor Eval 側: 運営者 1 名 + 必要に応じて学位機関関係者 1 名（credibility）
- 議連側: 事務局長もしくは秘書（議員本人が同席する場合はプラス）

**面会時間**: 30-60 分

**準備物**:

- brief の紙版（参加者分）
- プロジェクト説明用スライド（5-10 枚程度）
- データ・分析方法に関する質疑への対応メモ

**面会での主な論点**:

1. プロジェクトの研究背景・学位機関での位置づけ
2. データソース（公開クレジット）と個人情報保護
3. 現状の主要知見（ジェンダー / 中堅流出 / 可視性格差）
4. 議連の関心領域との接点
5. 今後の可能性（定期 brief / 招待 / 共同提言）

**禁止事項**:

- 特定の政党・議員への肩入れ発言
- 「御社 (議連) こそが解決策」と思わせる誇大表現
- 具体的な政策立案への関与提案（分析提供までが責務）

### Step 5: 招待・ヒアリング

**状況**: 議連が Animetor Eval の work に関心を示し、勉強会の講師招待、または政策提言会議での報告を求めてきた場合

**Animetor Eval 側の判定基準**:

- [ ] 超党派議連であるか（特定政党に極端に偏っていないか）
- [ ] 講演内容が「労働者寄り政策提言」の色が強すぎないか（学術発表の域を超えていないか）
- [ ] 他の議連からも同様の招待があるか、または予定されているか（バランス確認）
- [ ] COI disclosure が必要な場合は事前対応したか（startup の business 利害等）

**招待受託時の留意**:

1. 講演資料は事前に議連に提出し、承認を得る（議論の余地のある内容について政治的解釈が乗っかることを避けるため）
2. 講演後、他の主要議連にも同内容のプレゼンテーションの機会を提供する
3. 講演記録（速記録・動画）が公開される場合、Animetor Eval はそれを自団体の成果として二次利用できることを確認

### Step 6: 共同政策提言（発展形）

**状況**: 複数の議連が同じ policy direction （例：クレジット公表義務化）で政策提言をしたい場合

**実装形式**:

- 政策提言書の共著（議員 + Animetor Eval）
- 当該提言書は全議連に並行して提供（特定議連向けではない形式で）
- publication との同時出版を検討（査読済論文の政策転換版）

**内容の constraints**:

- 政策提言は labor-first direction のみ（労働環境改善・機会格差是正）
- 使用者側利益への迎合は避ける
- 「実装の詳細は政策当局の判断」という留保をつける

## 4. 材料送付（Material Package）

### 4.1 短縮版 Brief

**ファイル**: `result/md/policy_brief_short.md` / `result/html/policy_brief_short.html`

**内容**: 2 ページ、以下の構成

- Page 1: プロジェクト概要 + 主要数値 5 個 (3-5行)
- Page 2: Policy implications (3-5 個) + 連絡先 + 詳細 brief reference

**出力タイミング**: `pixi run pipeline` 実行時、自動生成

**言語**: JA （EN 版は将来オプション）

### 4.2 詳細版 Brief

**ファイル**: `result/md/policy_brief.md` / `result/html/policy_brief.html`

**内容**: 10-15 ページ

- Background (業界構造・現状課題)
- Findings (労働流動性・ジェンダー・可視性格差)
- Policy implications (推奨施策 5-10 個)
- Data statement / Limitations

**送付条件**: 初回 brief 送付後、議連から「詳細版が欲しい」との要求があった場合、または面会時に提供

### 4.3 プロジェクト概要シート

**ファイル**: `docs/policy/project_overview_sheet.md` （今後作成）

**内容**: 1 ページ

- 名前 / 研究機関 / 研究テーマ
- 公式ウェブサイト
- アフィリエーション / COI disclosure
- 連絡先（メール / 電話）

### 4.4 技術付録（オプション）

**ファイル**: `docs/CALCULATION_COMPENDIUM.md` など既存技術 doc への参照

**送付条件**: 議員秘書が detail に関心を持った場合、「以下の technical document で詳細を説明している」という形で参照を示す

## 5. 会議参加時の Protocol

### 5.1 招待から参加確定まで

1. **招待メール受信** → 24 時間以内に受託可否を判定
   - 研究スケジュール / COI の判定
   - 他議連とのバランス確認
   - 学位機関への報告

2. **受託回答** → メール返信
   ```
   貴議連の勉強会へのご招待をいただき、ありがとうございます。
   日程：[確認] / 資料提出期限：[確認] / 講演時間：[確認]
   
   本講演は、Animetor Eval の分析成果の学術的報告であり、
   政党・特定議員への政治的支持を表明するものではないことを
   あらかじめご了承ください。
   ```

3. **資料提出** → 7 日前までに draft 提出
   - 講演スライド PDF (日本語)
   - 英語版オプション（国会での国際比較発表の場合）
   - footer に「Animetor Eval — Structural Analysis of Anime Industry」を明記

4. **資料承認** → 議連から feedback / approval 取得
   - 特に政治的・党派的な誤読の可能性がないか確認

### 5.2 当日の対応

**場所**: 国会議員会館など（議連側が手配）

**参加者**:

- Animetor Eval: 運営者 1 名 + 必要に応じて副支援者 1 名
- 議連: chairperson / 秘書等 + 議員複数名

**スケジュール例** (30 分講演 + 15 分 Q&A):

- 5 分: introduction by 事務局長
- 20 分: Animetor Eval からのプレゼン
- 10 分: スライド / 追加補足
- 15 分: 質疑応答

**講演内容の縛り**:

- **許可**: 定量的 findings、データソース説明、限界明示
- **禁止**: 「○○議員のご指摘の通り」など特定議員への肩入れ、政策提言への直接的な言及

**Q&A での対応**:

| 質問の種類 | 対応 |
|-----------|------|
| 技術的質問（methodology） | 詳しく説明。CALCULATION_COMPENDIUM 等への参照 |
| 政策的質問（「～すべきでは」） | 「当プロジェクトは分析成果の提供であり、政策決定は貴議連と政府の判断」と明示 |
| 党派的質問（「○○党と比較して」） | 「政治的比較は当プロジェクトの範囲外。公開データに基づく業界分析に限定している」 |
| 将来の協力可能性 | 「学位論文・研究スケジュール内で可能な限り支援する」と曖昧でない返答 |

### 5.3 後日対応

1. **議事録・速記録の確認** → 出版前に誤記・誤引用がないか確認できることを事前協議で定める

2. **他議連への同内容プレゼン提供**

   議連 A で講演した場合、1-2 週間後に議連 B にも同等の機会を提供する。形式は異なってもよい（勉強会 vs. ヒアリング等）。

3. **成果報告** → 学位機関への報告、publication への該当資料の記載

## 6. 政治的中立性を破る事象への対応

以下のケースが発生した場合の対応フロー。

### ケース A: 特定議員から「我が党の選挙キャンペーンで使いたい」

**対応**:

1. 丁寧に感謝を述べつつ、「政治的中立性の原則上、単一政党の選挙活動への専属利用はできない」と説明

2. 「代わりに、全議連に同等の情報を提供しており、そちらの枠組みでご活用いただきたい」と提案

3. 回答期限を設定（「来週中にご返答いただくと幸いです」等）

4. 記録を残す（メール返信の cc: に学位機関の指導教員等を含め、透明性を確保）

### ケース B: 報道が「Animetor Eval は○○党寄りである」と報じた

**対応**:

1. **事実確認**: 実際に特定政党への偏向があったか検証（アクセスログ / 提供物の記録）

2. **事実なし の場合**:
   
   公式コメント発表（テンプレートは §2.3 参照）
   
   報道機関への訂正申し入れ（可能な範囲で）
   
   学位機関への報告

3. **事実ありの可能性がある場合**:
   
   内部調査：誰がどのような判断で特定議連を優先したのか
   
   学位機関との協議の上で、適切な謝罪と改善策を発表

### ケース C: 特定議連が「Animetor Eval の分析を党の政策立案に組み込みたい」と正式依頼

**判定基準**:

- 当該議連が超党派か（YESなら進める）
- 他の議連にも同条件を提供するか（YESなら可）
- 政策立案への関与が学位論文の範囲内か（検査）
- COI 開示は十分か（検査）

**条件付き受託の場合**:

```
貴議連の政策立案へのご協力をいただくことに感謝いたします。
以下の条件で参加させていただきたいと思います。

1. 本協力は Animetor Eval プロジェクトの研究活動の一環です
2. 同等の条件で他議連にも情報提供できることを前提とします
3. publication との sync を取ります（共著論文の可能性も含む）
4. 最終的な政策決定は貴議連・政府の判断であり、
   Animetor Eval の見解を代表するものではないことを
   公式に明記していただきたいです
```

## 7. Open Questions（User Judgment Pending）

以下の項目は、Animetor Eval の政治的・倫理的立場に関わるため、**ユーザーの確定判断待ち**である。

### Q1: 野党議連と与党議連への提供タイミング

**問題**: 政権交代時、与党→野党に転じた議連の扱い。新たに野党化した議連に先行提供すべきか、それとも現与党に合わせるべきか？

**現在の方針**: 党派横断的 / 時間遅延少なく平等提供（具体的な判断基準は user 確認待ち）

### Q2: 地方議会への展開

**問題**: アニメ産業が集中する東京都・京都府等の地方議会や地方議連への提供。国会議員連盟と同等の protocol を適用すべきか？

**現在の方針**: 未定。国会版が confirm された後に設計

### Q3: 海外議員・国際機関への提供

**問題**: 例えば EU 議会の creative industry committee、国連 UNESCO 等への提供。外交的・国益的判断が関わる場合の protocol は？

**現在の方針**: 未定。学位機関・外交官等との相談が必要

### Q4: 労働組合・経営者団体経由のアクセス

**問題**: 議連ではなく、JAniCA（労組）や映像制作委員会（経営側）からの議員紹介依頼。labor-first なので前者は優先度高いが、中立性をどう保つ？

**現在の方針**: labor-first stance からは前者優先を示唆するが、明示的な優先度基準は user 確認待ち
