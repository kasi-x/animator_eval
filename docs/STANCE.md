# STANCE.md — Animetor Eval プロジェクト・スタンス

本文書は Animetor Eval プロジェクトの **戦略的・規範的スタンス** を明文化する。
レポート執筆規範は `docs/REPORT_PHILOSOPHY.md`、設計原則は `CLAUDE.md`。本文書はそれらの上位にある「立場」の宣言。

最終更新: 2026-05-06。改訂履歴は文末。

---

## 0. なぜ STANCE.md が必要か

Animetor Eval は中立な記述基盤ではなく、**労働者寄り (labor-first) の構造観察プロジェクト** として設計される。
スタンスを明示しなければ、設計判断 (誰のために何を可視化するか / 何を可視化しないか) が場当たりになり、
パートナー (JAniCA / 個別アニメーター / スタジオ HR / 政策担当 / 学術) との関係も整合性を失う。

REPORT_PHILOSOPHY.md は「客観性の条件を選択の隠蔽から開示に置き換える」と書く。
本文書はその選択そのものを開示する: **誰のために、何を、どの目的で可視化しているか**。

## 1. Labor-first framing — 中核スタンス

### 1.1 立場宣言

本プロジェクトは、アニメ業界の労働者 (アニメーター・監督・制作・声優ほか個人クレジット保有者) の側に立つ。
中立を装わず、構造観察の成果が**労働者の権利可視化と報酬交渉の根拠**として使われることを意図する。

具体的に、以下を本プロジェクトの目的とする:

1. 個人クレジットの **集約と検索可能化** — 自分の貢献が業界全体でどう位置づくか、本人と第三者が確認できる
2. 構造的位置 (協業密度・ネットワーク中心性・役職進行) の **定量的可視化** — 報酬交渉時の客観的根拠
3. **「会社にクレジット公表を依頼する」根拠** — 「他のスタジオは公開している」「業界平均の可視性はこれだけある」を示せる比較データ
4. 機会格差 (gender / cohort / 所属) の **構造的検出** — 個人の能力差ではなく、システム的な機会偏在の証拠

### 1.2 スタンスがしないこと

- スタジオ・配信プラットフォーム・委員会への迎合的なレポート設計
- 「業界全体のため」を名目にした労働者保護の後退
- 中立を装ったまま使用者側の利便 (低コスト人材選別等) に資する設計

### 1.3 既存ハードルールとの整合

- **H1 (anime.score 不使用)**: 維持。viewer rating は労働者評価の根拠にできない
- **H2 (能力 framing 禁止)**: 強化。個人を「能力で序列化」する frame は労働者寄り立場と相容れない
- **H3 (entity resolution 不変)**: 維持。誤マッチは労働者の信用毀損に直結
- **H4 (補償根拠は analytical CI)**: 強化。労働者寄りスタンスを取るからこそ、数値の頑健性に妥協しない

## 2. プロジェクト目的経路

3 経路を以下の重み付けで併走する (2026-05-06 ユーザー決定):

| 経路 | 重み | 役割 |
|------|------|------|
| Publication | 59% | 査読耐性のある evidence 生産。経済学 + 情報科学の両分野。RIETI DP / Labour Economics / J. Cultural Economics / NetSci 系 |
| 政策提言 | 32% | 経産省コンテンツ産業課・文化庁メディア芸術課・業界団体・議連経由の policy brief。労働者保護 / 機会格差是正の方向 |
| Business | 9% | スタートアップとしての実装基盤。pilot / SaaS は publication / 政策の **実装・露出** チャネル。短期収益化を主目的にしない |

Business 比率が低い理由: 短期収益化を最優先すると顧客 (スタジオ HR) の利害に引きずられ、labor-first スタンスと緊張する。
代わりに **個人アニメーター向け B2C** や **労働者団体 (JAniCA) 向けデータ提供** の方向で business 経路を運営する。

## 3. 個人情報・公開設計の原則

### 3.1 「公開クレジット」の解釈

- 公開クレジット = エンドロール / 公式 staff list / 公開データベース (AniList / MAL / ANN / TMDb / bangumi 等) で **既に公開されている情報**
- これらの **集約と分析** は新たな公開ではなく、既公開情報の構造化として扱う
- 集約自体は GDPR / 個人情報保護法上「新たな処理」に該当しうるが、**公正な集約 + 当事者の opt-out 受付** で運用する

### 3.2 年齢 / DOB の扱い

- エンドクレジットに年齢情報は含まれない。仕事をしている時点で「労働者」であり、未成年保護対象としての特別扱いはしない
- ただし以下を遵守:
  - **DOB を public display 層に出さない** (現状: routers / static frontend に流入なし、維持)
  - **年齢推論を scoring に使わない** (cohort 定義に debut 年使用は可、生物学的年齢は不可)
  - **DOB を保持する schema 列** (`persons.date_of_birth` 等) は entity resolution / cohort 推定の internal use のみ

### 3.3 故人の扱い

- 故人クレジットは scoring / 集計に通常通り含む
- display 層では「故人」表示の有無を **将来検討** (現状未対応、urgent ではない)

### 3.4 ペンネーム / 匿名活動者

- penname は entity として独立に扱い、本名と紐付けない
- 本人申請がない限り penname と本名の merge は行わない

### 3.5 opt-out 機構

- **on-request 削除** を最低限の保証とする (公開後 7 日以内対応)
- 削除リクエストは個人ページ + メール窓口で受付
- 削除後も集計 (匿名化済) からは復元不能なよう設計

## 4. ステークホルダーとの関係

### 4.1 JAniCA (アニメーター労組)

- **事後設計**: JAniCA からの要求が来たら合わせる方針 (先回りして protect しない)
- **連携志向**: 労働者寄りスタンスは JAniCA と整合的、partner 化を視野
- データ共有 / 政策提言の共同実施を検討

### 4.2 個別スタジオ

- 大手スタジオ HR を顧客とする B2B SaaS は当面しない
- pilot は「労働者寄りで運営される」ことを了解するスタジオに限定
- 通常スタジオには「業界レポート閲覧」レベルのアクセスのみ

### 4.3 配信プラットフォーム (Netflix / Crunchyroll 等)

- 顧客化はしない。データ需要があっても、労働者寄りスタンスと利害が衝突しうる
- ただし業界レポート閲覧・取材対応は通常通り

### 4.4 個別アニメーター

- 中核ユーザー。**自分の構造的位置を自分で確認できる** B2C SaaS が最終形
- SNS 経由で interview / フィードバック収集
- 公開前 review 機構: opt-out のみ (本人 review 機能は将来検討)

### 4.5 政府 / 政策担当

- METI コンテンツ産業課 / 文化庁メディア芸術課 / 業界団体 / 議連 経由
- brief は 2 ページ短縮版 (現状の長文 Policy brief とは別)
- 政策推奨は labor-first 方向 (機会格差是正・クレジット公表義務化検討)

### 4.6 メディア / 業界誌

- reactive (取材依頼が来たら受ける)
- publication 公表時のみ proactive プレス release を検討

## 5. 学術スタンス

### 5.1 投稿先 (multi-track)

- **経済系 (主)**: 日本経済学会 (JEA) / Labour Economics / Journal of Cultural Economics / RIETI Discussion Paper
- **情報・ネットワーク系 (副)**: Applied Network Science / NetSci / WebSci

### 5.2 経済 + 情報DS 両建ての論文戦略

ユーザーが両分野の専攻を持つため、以下のいずれか:

- (a) 1 論文 1 venue (経済論文と情報論文を別々、最も安全)
- (b) 経済主・情報副 (経済 venue 主投稿、情報側は workshop / poster)
- (c) 情報主・経済副 (逆)
- (d) 同データ・別 cut で 2 論文 (倫理・自己引用注意)

→ 詳細は `TASK_CARDS/32_publication/00_paper_strategy.md` で確定。

### 5.3 機関 affiliation

ユーザーは社会人博士課程院生として既に機関 affiliation を持つ。客員研究員等の追加ポジション獲得は不要。
本プロジェクトは **学位論文と独立した個人 / startup プロジェクト**、指導教員の関与なし。

### 5.4 Replication policy

- on-request (査読者・申請者にのみ提供) を基本
- ただし高位 venue (Labour Economics / J. Cultural Economics) 投稿時のみ snapshot 凍結 + Zenodo DOI 付与の **exception** を認める
- 詳細は `TASK_CARDS/32_publication/01_replication_snapshot_exception.md`

## 6. Reflexivity (Goodhart リスク)

### 6.1 認識

スコア化 → 業界がスコアを目的化するリスクは存在。「ネットワーク位置」も実質「業界内権力」を可視化する。
完全な回避は不能。

### 6.2 対応

- **REPORT_PHILOSOPHY.md §8** で言及済 (現状維持)
- 各レポートに caveat 段落を入れる (将来検討、現状未対応)
- **専用 reflexivity doc は作らない** (現フェーズでは過剰)
